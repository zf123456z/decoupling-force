# 架构设计示例：AI Agent 认知引擎

## 场景

设计一个 AI Agent 的认知引擎。传统做法是"一个引擎做所有事"——记忆、推理、响应都在同一套机制里。

这会导致：加新功能困难、推理和记忆互相干扰、性能瓶颈。

## 应用解耦力框架

### Step 1: 找裂缝

| 判据 | 分析 |
|:-----|:------|
| 时间尺度差异 | 长期记忆（月级）vs 短期推理（秒级）vs 响应生成（毫秒级） |
| 职责差异 | "记住" vs "思考" vs "表达" 是完全不同的认知功能 |
| 资源需求差异 | 记忆需要大容量存储，推理需要快速检索，响应需要流畅输出 |

**裂缝位置**：沿着"时间尺度"和"认知功能"的交叉边界。

### Step 2: 切割

```
单一认知引擎
  ├── 骨架层（Skeleton）  — 长期知识固化，FSQ式离散节点
  ├── 皮层层（Cortex）    — 临时推理，上下文感知
  └── 桥接层（Bridge）    — 骨架→皮层的按需投影
```

### Step 3: 桥接

- 骨架层暴露检索接口（query → top-K 相关节点）
- 皮层层在每次推理时通过桥接层拉取骨架知识
- 对话结束后皮层释放，骨架层持续积累
